// workflow

From raw page to verdict in three steps

Open a SIEM, a ticket, a CTI report or an email — Mustela does the rest.

step 01

IOC highlighted in place

Mustela scans the page and highlights every indicator it finds, without altering the host page's behavior.

step 02

One click, one panel

Click any highlight to open the in-page investigation panel with consolidated provider verdicts, notes and JSON export.

step 03

Your investigation trail

The popup summarizes the current page, offers manual lookup, and keeps your recent investigations at hand.

// trust posture

Transparent by design

No backend, no telemetry, no analytics, no account system. Mustela is honest about its trade-offs.

  • Detection runs locally — IOC parsing and highlighting never leave the browser.
  • Storage stays local — settings, cache, history and notes live in chrome.storage.local.
  • Lookups are explicit — an IOC is only sent to a provider you enabled, when you ask for it.
  • Documented trade-offs — full details in PRIVACY_TRANSPARENCY.md.
// deployment

Up and running in two minutes

Works out of the box for detection and pivots. Add free API keys for enriched verdicts.

# build from source
git clone https://github.com/ArmorIntel/mustela.git
cd mustela
npm install && npm run build
  1. Open chrome://extensions and enable Developer mode
  2. Click Load unpacked and select dist/chrome
  3. Optional: add free VirusTotal / AbuseIPDB / Shodan keys in Options
